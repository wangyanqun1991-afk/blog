关键发现：VSCode 扩展的 settings.json 不支持 mcpServers 字段。正确做法是在项目树中添加 .mcp.json 文件
我自己的json位置是 C:\Users\admin\.claude\settings.json 文件

1.需要在settings.json 中  "enabledMcpjsonServers": 这是开启对应的MCP服务
2.因为插件不支持直接在这个setting.json里写mcpServers
3.所以应该在项目目录中 写一个.mcp.json 然后配置mcpServers字段的MCP
4.本地项目中的 .mcp.json 只针对该项目,而不是全局

配置文件settings.json 有四个位置：

    作用域	        文件路径
    项目（共享)   	.claude/settings.json
    项目（本地）	    .claude/settings.local.json
    用户（共享）	    ~/.claude/settings.json
    用户（本地）	    ~/.claude/settings.local.json

我自己的json位置是 C:\Users\admin\.claude\settings.json 如上,这是一个用户级的文件,deepseek的env 环境配置都在这里面

对，但不限于项目目录，有两个位置可以放：

位置	路径	作用范围
项目级	<项目根>/.mcp.json	仅当前项目
用户级	~/.mcp.json（即 C:\Users\admin\.mcp.json）	所有项目通用
两种都需要在 settings.json 中用 enabledMcpjsonServers 批准才能启用，这是一个安全机制，防止项目中的 .mcp.json 未经你同意就启动外部进程

 官方 Claude Code 扩展自带了一个 claude.exe 在扩展目录里。用这个来添加 deepwiki。
这是


1. .claude.json 是什么
它是 Claude Code 的全局用户配置文件。之前加的 deepwiki 就在里面。不同于 .mcp.json（项目级，共享给团队），这个文件是你个人电脑上私有的，存的是你自己的设置。

2. 自带的 claude.exe 可以用命令行启动
对，就在这个位置：


C:\Users\admin\.vscode\extensions\anthropic.claude-code-2.1.111-win32-x64\resources\native-binary\claude.exe
不过你刚才也看到了，在 Windows 上直接运行它需要指定 git-bash 路径:


set CLAUDE_CODE_GIT_BASH_PATH=D:\Git\usr\bin\bash.exe
"C:\Users\admin\.vscode\extensions\anthropic.claude-code-2.1.111-win32-x64\resources\native-binary\claude.exe"
如果你想让 claude 命令直接在终端生效（比如直接打 claude mcp list），有两种方式:

加到 PATH: 把 C:\Users\admin\.vscode\extensions\anthropic.claude-code-2.1.111-win32-x64\resources\native-binary\ 加到系统环境变量
设 CLAUDE_CODE_GIT_BASH_PATH: 把 D:\Git\usr\bin\bash.exe 设为系统环境变量，这样每次不用手动指定
